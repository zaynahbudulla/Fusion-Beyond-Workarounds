Section 01

The Gap Oracle R12 Left Behind

Consignment inventory — where a supplier delivers goods but retains ownership until consumption — offers real cash flow and supply chain advantages. Yet Oracle R12 never gave it the native treatment it deserved. The result was a patchwork of manual processes, custom development, and uncomfortable workarounds.

Oracle R12 The old reality
❌ No native consignment agreement structure — simulated via blanket POs
❌ Consumption tracked manually in spreadsheets or external tools
❌ No distinction between owned and consigned stock in inventory
❌ Invoicing handled outside standard AP — prone to errors and delays
❌ Returns managed ad hoc with no standard flow or traceability
❌ Manual journal entries required for correct accounting treatment
Oracle Fusion The new standard
✅ Formal Consignment Agreements as a dedicated document type
✅ Consumption tracked automatically with system-generated advice
✅ Consignment sub-inventory clearly tagged as supplier-owned
✅ AP invoices auto-generated from consumption events
✅ Full native return flow with linkage back to originating order
✅ Subledger Accounting (SLA) handles all entries automatically

Section 03

Returns: Finally a First-Class Citizen

In R12, returning unconsumed consignment goods was an operational headache with no standard solution. Oracle Fusion treats returns as an integral part of the consignment flow, not an afterthought.

Consignment Sub-Inventory
→
Return Transaction Initiated
→
Linked to Original Order
→

→
Supplier Credit / No Invoice

The system ensures that only goods confirmed as consumed are billed. Unconsumed returns are tracked back to the supplier with full documentation — no disputes, no manual corrections, no data integrity issues.
